
## 补码

在硬件层面，加法器比较容易实现。因此考虑“补码”操作，将减法运算通过某些比较容易实现的方式，转变为加法运算。

example：

```
62   #十进制
0 0111110  #62的二进制
1 1000001  #取反码
1 1000010  #加上1，得到-62的补码   
```

```
    0 0111110  (+62)
+   1 1000010  (-62) #符号位也参与运算
= 1 0 0000000        #最高位的1应被舍弃，因为是8位的表示，溢出了

# -61的补码为1 1000011
```

注意，4Bytes表示一个int（1位为符号位），由于补码操作，其表示范围为-2^31 ~ 2^31-1
